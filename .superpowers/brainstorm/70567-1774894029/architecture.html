Key Design Choices

Sessions
Sessions become agent-agnostic. The router picks the agent per-message. Agent stored on each message, not the session.
Streaming
Router response is instant (small LLM call). Agent response streams via SSE. Client sees agent badge appear, then streamed text.
Wiki widget
Existing agent-page widget stays. It bypasses the router (agent is known). Central chat uses the graph.
Auth
Same MW cookie auth. User must be logged into wiki. /chat page checks session, redirects to wiki login if needed.
